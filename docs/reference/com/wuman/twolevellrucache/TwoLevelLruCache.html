Class Overview

A two-level LRU cache composed of a smaller, first level LruCache in memory and a larger, second level DiskLruCache. The keys must be of String type. The values must be convertible to and from a byte stream using a Converter.

Protected Methods

protected V create (String key)

Called after a cache miss to compute a value for the corresponding key. Returns the computed value or null if no value can be computed. The default implementation returns null.

The method is called without synchronization: other threads may access the cache while this method is executing.

If a value for key exists in the cache when this method returns, the created value will be released with entryRemoved(boolean, String, V, V) and discarded. This can occur when multiple threads request the same key at the same time (causing multiple values to be created), or when one thread calls put(String, V) while another is creating a value for the same key.

Returns
  • createdValue

protected void entryRemoved (boolean evicted, String key, V oldValue, V newValue)

Called for entries that have been evicted or removed. This method is invoked when a value is evicted to make space, removed by a call to remove(String), or replaced by a call to put(String, V). The default implementation does nothing.

The method is called without synchronization: other threads may access the cache while this method is executing.

Parameters
evicted true if the entry is being removed to make space, false if the removal was caused by a put(String, V) or remove(String).
newValue the new value for key, if it exists. If non-null,this removal was caused by a put(String, V). Otherwise it was caused by an eviction or a remove(String).

protected int sizeOf (String key, V value)

Returns the size of the entry for key and value in user-defined units. The default implementation returns 1 so that size is the number of entries and max size is the maximum number of entries.

An entry's size must not change while it is in the cache.
